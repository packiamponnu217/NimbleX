# NimbleX — Business & Product Documentation

## 1. Company Overview

### Brand Name
NimbleX

### Domain
[nimblex.dev](https://nimblex.dev)

### Tagline
Building the next generation of software engineers.

---

## 2. Vision
To become a platform that transforms students into industry-ready software engineers through structured learning, practical execution, mentorship, and real-world product development.

---

## 3. Mission
NimbleX aims to solve the biggest challenge faced by college students:
**The lack of a clear roadmap and practical industry experience.**

Many students learn programming and technologies without understanding:
* What to learn
* Why to learn
* How companies build software
* How to prepare for jobs

NimbleX bridges this gap by providing:
* Structured engineering roadmaps
* Daily problem-solving practice
* Industry-level development workflows
* Real product experience
* Continuous mentorship
* Career preparation support

---

## 4. Core Philosophy

### Traditional learning:
`Learn` $\rightarrow$ `Complete Course` $\rightarrow$ `Search for Job`

### NimbleX approach:
`Learn` $\rightarrow$ `Practice` $\rightarrow$ `Build` $\rightarrow$ `Deploy` $\rightarrow$ `Experience Industry Workflow` $\rightarrow$ `Become Job Ready`

## 7. Software Engineering Cohort Journey

### Month 1 — Fundamentals & Problem Solving

#### Objective
Build strong programming foundations and engineering thinking.

#### Students learn:
* Programming fundamentals
* Logical thinking
* Problem-solving approach
* Coding practices

#### Daily Practice
Students solve **2 coding problems daily**.
* **Purpose:**
  * Improve logical ability
  * Build consistency
  * Develop an engineering mindset

---

### Month 2 — Frameworks & Software Development

#### Objective
Transition students from programming learners into software developers.

#### Students learn:

##### Backend
* FastAPI
* Fastify
* ORM

##### Frontend
* React
* Frontend-backend connectivity

#### Development Process
Students follow:
`Define` $\rightarrow$ `Design` $\rightarrow$ `Develop`

They learn:
* Requirement understanding
* Solution design
* Implementation

#### Final 3 Days of Month 2 (Industry Practices)
* Git
* CI/CD
* Cloud DevOps basics
* Deployment workflow

---

### Month 3 to Month 7 — Product Engineering

#### Objective
Provide real software engineering experience.

#### Students work on:
* Live software projects
* Real product development
* Team-based engineering

#### Engineering Workflow
Students follow: **Sprint-Based Development**
* **Process:** `Requirement` $\rightarrow$ `Issue Creation` $\rightarrow$ `Development` $\rightarrow$ `Review` $\rightarrow$ `Deployment`
* **Tools:**
  * GitLab Issues
  * Version Control
  * Project Management Workflow

  * Help students overcome blockers

### Project Manager Support
Each project has a Project Manager.
* **Responsibilities:**
  * Sprint planning
  * Task assignment
  * Team coordination
  * Technical guidance
  * Project execution support

## 10. NimbleX Differentiation
NimbleX is different because:

1. **Roadmap Based Learning:** Students know exactly what to learn.
2. **Daily Coding Practice:** Students build problem-solving ability.
3. **Real Product Experience:** Students work like software engineers.
4. **Industry Workflow Exposure:** Students experience Git, Issues, Sprints, CI/CD, and Deployment.
5. **Personal Mentorship:** Students receive continuous guidance.
6. **Career Transformation:** The goal is not just completing a course, but creating confident, industry-ready software engineers.
